# Stage Lab 练习页全身锁定舞台设计

**状态：** 已确认，等待实施计划

## 目标

重做 iPhone 17 Pro Max（iOS 26.5）练习页，使视频练习成为首屏主角，同时清晰呈现所选舞者、完整全身构图、最小播放器控制与下一动作。修复当前页面中视频层、目标框、骨架和下方控制区互相竞争、文字被裁切的问题。

## 范围

包含：

- `PracticeView` 与练习视频舞台的 SwiftUI 重新布局。
- 9:16 视频占首屏约 84% 的沉浸舞台。
- 所选舞者的完整全身锁定构图与稳定回退呈现。
- 紧凑视频内控制栏、动作节点摘要和低干扰骨架叠加。
- iPhone 17 Pro Max（iOS 26.5）Simulator 的视觉与交互验收。

不包含：

- 后端、模型、Analysis Package schema、云资源或 iOS 27 适配。
- 新的舞者识别、动作纠错或音乐语义功能。
- 真实视频进入 Git、改变隐私或数据保存策略。

## 信息层级与布局

1. 顶部导航保持系统安全区内的返回与“练习”标题；不在视频画面上追加大标题。
2. 视频舞台位于首屏主体，高度约占可视页面的 84%。视频内容保持 9:16，圆角只施加于视频容器，不让任何文字或控制区被裁切。
3. 视频左上只显示一个半透明状态胶囊：`舞者 N · 全身锁定`。当无可用跟随数据时，分别显示 `完整画面` 或 `跟随数据不可用`，不得暗示仍在锁定目标。
4. 视频底部依次放置标题/时间、3pt 进度条和 45pt 半透明控制栏。控制栏只包含圆形播放/暂停、镜像、循环、骨架和速度胶囊；按钮最小可点击区域为 44pt，即使图标视觉尺寸更小。
5. 视频后仅显示紧凑的“下一动作”卡片。完整的时间轴和辅助设置继续放在滚动页下方，不能挤入视频层。

## 所选舞者全身锁定

- 使用现有 spotlight track 和姿态轨迹构建 9:16 跟随合成；构图目标是完整身体安全区域，而不是追随单一身体点。
- 有效关键帧须保留头部、脚部和横向动作余量。平滑变化仅在相邻可信关键帧之间产生；不因单帧框抖动而跳切。
- 跟随数据短暂不可用时保持最后一个可信全身构图；无法安全保持时退回完整原始画面。回退状态必须与画面一致。
- 移除所有黄色目标框、边界矩形和“检测框”视觉。目标识别仅由状态文字表达。

## 骨架与视觉样式

- 骨架默认关闭。开启后仅在视频坐标系绘制，随镜像和跟随裁切同步。
- 骨架采用低透明度青色关节点与人体相邻骨段，不得把按数组排序的点直接串成折线。
- 视频使用深靛/中性遮罩保证白色元数据可读；不得通过整体绿色滤镜、厚重渐层或黄色框遮挡舞者。
- 控制栏使用深色材质与轻描边；下方内容区使用浅色面板，形成明确的舞台/信息层分界。

## 状态与错误处理

- 本地视频不存在时继续显示现有“视频不可用”恢复页面；所有运输控制禁用，不能显示伪播放状态。
- 无 Analysis Package 时播放器仍可用，状态显示“跟随数据不可用”，骨架不渲染。
- 无可靠全身构图时显示“完整画面”，不显示任何目标边界或过期骨架。
- 速度、镜像、循环、进度与离开页面清理保留现有行为；重新布局不得改变其持久化语义。

## 验收标准

- iPhone 17 Pro Max（iOS 26.5）Simulator 上，视频成为首屏主体；控制栏、标题、时间和下一动作均完整可读且不越界。
- 正常跟随时无黄色框，状态为“舞者 N · 全身锁定”；目标全身在头、脚和两侧余量内稳定可见。
- 骨架默认关闭；开启后只显示低干扰人体骨段，且镜像与裁切对齐。
- 完整画面与无数据回退状态的文字、视频与 overlay 一致。
- 现有完整 iOS 测试、Debug 构建与新增布局/投影单元测试通过。真实样本的播放、镜像、速度、循环与离线重启仍须单独记录，不以模拟数据替代。

- 使用真实本地样本时记录素材不进入 Git、实际可执行检查和未执行项目。

- 现有 Simulator 项目可能缺少本地媒体，因此它只能验证布局和恢复态；完整播放验证需要重新导入本地样本。
- 不修改 Analysis Package schema 或后端。若新舞台出现问题，可回退本次 UI 提交，原有播放器数据与项目状态保持不变。
