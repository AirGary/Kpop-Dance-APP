# 竖屏全身跟随练习页设计

## 目标

让本地真实 AI Demo 的练习页成为可用于跟跳的竖屏体验：选择目标舞者后，画面以 9:16 显示，自动平滑跟随该舞者，同时优先保留完整身体动作。

## 范围

- 测试设备固定为 iPhone 17 Pro Max、iOS 26.5。
- 练习页播放器显示区固定为竖屏 9:16，播放、速度、镜像和循环控制保留在视频下方。
- App 消费 Analysis Package 现有的 `spotlight-track.json`，对目标舞者生成带安全边距的裁切窗口。
- 视频画面、骨架、人物框和镜像必须使用同一套裁切变换。
- 目标丢失、被遮挡、轨迹超出画面或置信度过低时，平滑回退到不锁定主体的完整画面，不能突然跳变或强行放大。
- 始终分析、保存和验证目标舞者全身姿态关键点与轨迹。当前练习页可以显示全身骨架；以后手部或腿部局部特训只在 App 本地筛选和放大已有全身数据，不重新上传或运行 AI。

## 不包含

- 不改 RTMDet-m、ByteTrack、RTMPose-m 模型组合，不新增模型训练。
- 不改 Analysis Package schema、后端 API、本地 FastAPI 或 Google Cloud 资源。
- 不处理 iOS 27 或其他设备适配。
- 不实现手部、腿部等局部特训 UI；仅保证其未来所需的全身数据边界。

## 方案

### 画面构图

默认采用“全身练习跟随”：裁切窗口目标比例为 9:16，围绕当前目标框扩展安全边距，且优先使头、双手和双脚保留在画面内。窗口在源视频边界内裁剪；无法构成可靠全身窗口时不再继续放大。

### 跟随算法

App 按播放时间在相邻 `AnalysisSpotlightKeyframe` 之间线性插值。每帧把归一化人物框转换为 9:16 裁切窗口，并在连续窗口间应用平滑过渡，减少跟随抖动。关键帧距离当前时间过远、置信度低于阈值或窗口无效时进入回退状态，平滑过渡到完整画面。

### 渲染边界

使用可控制内容位置与缩放的播放器承载视图替代当前只能居中裁切的 `VideoPlayer`。渲染层接收当前裁切矩形；分析叠层使用相同坐标变换绘制骨架和人物框。镜像在最终组合后统一应用，以避免视频和叠层方向不一致。

### 全身数据与未来局部训练

`pose-track.json` 保持完整目标姿态帧。局部训练未来只能以全身骨架的命名关键点为输入，在本地选择身体区域、计算建议片段和调整裁切；不会改变已保存的原始分析结果，也不会要求云端重新计算。

## 错误处理

- 无结果包、缺少轨迹或视频尺寸未知：继续以完整画面播放，并显示不阻断的“跟随数据不可用”状态。
- 低置信度或遮挡：回退完整画面；恢复稳定轨迹后再平滑进入跟随。
- 非法坐标、NaN、超出边界的框：忽略该帧，不能让播放器崩溃或产生越界变换。

## 验收与测试

- 单元测试：9:16 裁切比例、边界裁剪、安全边距、关键帧插值、低置信度回退、镜像后叠层一致性、全身关键点保持可用。
- iOS 构建与测试：仅指定 iPhone 17 Pro Max（UDID `DF1DB4C3-C579-46A7-8BEB-D1B01D99A7DA`）和 iOS 26.5。
- 本地真实视频冒烟：上传、选定一名候选舞者、进入练习页；确认跟随窗口平滑、全身优先、骨架与人物一致、速度/镜像/循环可用。
- 回归：Analysis Package 解码、结果包哈希校验和本地离线练习不能退化。

## 回退

此功能只改变 iOS 端渲染。出现问题时可关闭跟随渲染并继续使用完整画面播放器；不删除结果包、不修改原始视频、不影响本地或云端分析任务。
