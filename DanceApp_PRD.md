# K-pop 舞蹈自学 App 产品需求文档

## 1. 产品概述

### 1.1 产品名称

暂定名：K-pop Dance Learning App

### 1.2 产品定位

一款面向 K-pop 舞蹈学习者的 iOS 自学工具。用户可以上传固定机位舞蹈视频，选择想学习的成员或站位，App 通过 AI 分析舞蹈动作、节拍、难点和队形变化，帮助用户更高效地拆解、练习和复盘。

### 1.3 核心价值

- 把复杂的 K-pop 舞蹈视频拆解成更容易学习的动作节点。
- 帮助用户识别高难动作、快速手部动作、腿部动作和转场。
- 支持按成员站位学习，降低扒舞和跟练成本。
- 通过慢速、节拍、镜像和分段练习提升自学效率。

### 1.4 示例使用场景

用户想学习 XG《GALA》舞蹈：

1. 上传或导入固定机位舞蹈视频。
2. 选择自己想学习的成员位置。
3. App 自动识别该成员的动作轨迹、关键动作和节拍点。
4. 用户用 0.5x 或 0.75x 慢速练习难点段落。
5. 用户开启镜像模式和节拍练习模式，对齐动作和音乐。

## 2. 目标用户

### 2.1 主要用户

- K-pop 翻跳爱好者
- 舞蹈初学者
- 舞蹈社团成员
- 想自学指定成员动作的粉丝
- 需要快速扒舞、分段练习的人

### 2.2 用户痛点

- 官方舞蹈视频动作快，难以看清细节。
- 多人舞蹈中很难持续跟踪某一个成员。
- 不知道哪些动作是关键动作，哪些只是过渡。
- 难以判断动作是否踩在正确节拍上。
- 队形变化复杂，容易记错站位和移动路线。
- 自学时缺少老师帮忙拆动作和标重点。

## 3. 产品目标

### 3.1 MVP 目标

在 iOS 上完成一个可用的 K-pop 舞蹈学习闭环：

- 用户可以导入视频。
- 用户可以选择视频中的舞者。
- App 可以提取舞者身体关键点。
- App 可以生成基础动作节点和节拍节点。
- 用户可以分段、慢速、镜像练习。

### 3.2 中长期目标

- 提供更准确的 AI 动作拆解。
- 支持动作评分和用户自拍视频对比。
- 支持队形学习和移动路线可视化。
- 建立 K-pop 舞蹈学习内容库。
- 形成免费加订阅的商业模式。

## 4. 平台与技术栈

### 4.1 平台

- iOS only
- 面向 App Store 发布
- 初期不考虑 Android、Web 或桌面端

### 4.2 开发工具

- Xcode
- Swift
- SwiftUI
- Claude AI 辅助开发

### 4.3 AI 与媒体分析技术

- iOS Vision framework：用于人体姿态识别，相当于 iOS 内建的 MediaPipe 替代方案。
- AVFoundation：用于视频播放、裁剪、帧处理和速度控制。
- Accelerate / SoundAnalysis / 自定义音频处理：用于基础节拍、BPM 和音频特征分析。
- librosa 等价能力：不直接在 iOS 内使用 librosa，而是在 Swift 侧实现或通过可移植算法完成 beat detection。

## 5. 核心功能需求

## 5.1 视频导入

### 功能描述

用户可以从本地相册导入舞蹈视频。初期主要支持固定机位视频，例如 dance practice、performance video 或 fancam。

### 用户故事

作为舞蹈学习者，我希望导入一段舞蹈视频，这样我可以在 App 中分段学习。

### 需求

- 支持从相册选择视频。
- 支持常见 iOS 视频格式。
- 导入后生成视频缩略图。
- 显示视频时长、分辨率和基本信息。
- 本地保存用户导入的视频记录。

### MVP 范围

- 只支持本地视频导入。
- 不支持 YouTube、TikTok、Instagram 在线链接解析。
- 不提供版权内容下载能力。

## 5.2 舞者识别与选择

### 功能描述

App 对视频画面中的人体进行姿态检测，识别多人舞者，并允许用户选择想学习的目标舞者。

### 用户故事

作为用户，我希望选择某一个成员的位置，这样我可以只学习这个人的动作。

### 需求

- 自动识别视频中的人体关键点。
- 支持多人画面中的舞者检测。
- 在视频画面上显示可选择的人物框或编号。
- 用户可以点击选择目标舞者。
- 选择后持续跟踪该舞者的动作轨迹。

### 风险与限制

- 人物遮挡会影响识别准确率。
- 镜头移动、低清晰度、强光或暗光会降低检测质量。
- 多人交叉换位时可能出现身份跟踪错误。

## 5.3 AI 动作拆解

### 功能描述

App 根据舞者姿态关键点和节拍，将舞蹈动作拆解为不同类型的学习节点。

### 节点类型

- Beat nodes：节拍节点，标记音乐中的主要节拍点。
- Key action nodes：关键动作节点，标记动作定格、重心变化、手势变化或视觉重点。
- Transition nodes：过渡节点，标记连接两个关键动作之间的移动过程。

### 用户故事

作为用户，我希望看到舞蹈中的关键动作节点，这样我可以先学最重要的动作，再练连接动作。

### 需求

- 根据音频检测 BPM 和节拍。
- 根据姿态变化识别动作变化点。
- 标记关键动作和过渡动作。
- 在视频进度条上显示节点。
- 点击节点后跳转到对应时间点。
- 支持按节点循环播放。

### MVP 范围

- 初期只做基础节点检测。
- 节点可允许用户手动调整或删除。
- 不要求 AI 拆解达到专业舞蹈老师级别。

## 5.4 难点检测

### 功能描述

App 自动识别视频中动作密度高、速度快、变化大的片段，并标记为高难片段。

### 识别指标

- 手部移动速度
- 腿部移动速度
- 身体重心变化
- 方向切换频率
- 单位时间内关键点变化幅度
- 节拍内动作数量

### 用户故事

作为用户，我希望 App 自动告诉我哪几段最难，这样我可以优先慢速练习。

### 需求

- 自动标记高难片段。
- 在时间轴上用不同颜色显示难度。
- 支持按难度筛选练习片段。
- 支持快速进入“只练难点”模式。

## 5.5 速度控制

### 功能描述

用户可以用不同播放速度学习舞蹈。

### 需求

- 支持 0.5x 播放。
- 支持 0.75x 播放。
- 支持 1x 播放。
- 切换速度时保持当前播放位置。
- 支持片段循环播放。

### 后续可扩展

- 0.25x 播放。
- 自定义速度。
- 保持音高不变的慢速播放。

## 5.6 镜像模式

### 功能描述

用户可以开启镜像模式，让视频方向更适合跟练。

### 用户故事

作为用户，我希望打开镜像模式，这样我可以像照镜子一样跟着跳。

### 需求

- 支持视频水平翻转。
- 镜像状态可随视频保存。
- 镜像模式不影响原视频文件。

## 5.7 节拍练习模式

### 功能描述

App 根据音乐节拍帮助用户踩点练习。

### 需求

- 自动检测 BPM。
- 在播放界面显示节拍提示。
- 在时间轴上显示主要节拍点。
- 支持跟随节拍循环练习。
- 可选开启节拍音或视觉提示。

### 后续可扩展

- 节拍计数：1, 2, 3, 4, 5, 6, 7, 8。
- 支持舞蹈常用八拍结构。
- 支持用户手动校准节拍偏移。

## 5.8 队形与站位学习

### 功能描述

App 分析多人舞蹈中的站位变化，帮助用户理解目标舞者在队形中的位置和移动路线。

### 用户故事

作为用户，我希望知道我学习的成员什么时候移动到哪里，这样我可以练习完整队形。

### 需求

- 显示目标舞者在画面中的位置轨迹。
- 标记明显换位节点。
- 提供简化的俯视队形图。
- 支持查看目标舞者移动路线。

### MVP 范围

- 初期可以只做 2D 画面位置轨迹。
- 不强制实现真实舞台空间还原。

## 6. 页面与时间逻辑

MVP 最低需要 5 个页面。队形与站位学习先不做独立页面，相关信息合并到练习页中展示。

## 6.1 页面清单

### 页面 1：首页 / 项目列表页

### 内容

- 最近导入的视频项目。
- 每个项目的分析状态。
- 新建学习项目入口。
- 订阅或高级功能入口。

### 页面状态

- 空状态：用户还没有导入任何视频。
- 有项目状态：展示已导入的视频项目列表。
- 项目未完成状态：显示分析中、待选择舞者或可继续练习。

### 跳转逻辑

- 点击“导入视频”进入视频导入页。
- 点击已完成分析的项目进入练习页。
- 点击分析中的项目进入视频分析页。
- 点击待选择舞者的项目进入舞者选择页。

### 页面 2：视频导入页

### 内容

- 本地相册视频选择入口。
- 已选择视频的基础信息。
- 创建学习项目按钮。

### 跳转逻辑

1. 用户从首页点击导入视频。
2. 系统打开相册选择器。
3. 用户选择视频。
4. App 创建学习项目。
5. 创建成功后进入视频分析页。

### 页面 3：视频分析页

### 内容

- 分析进度。
- 当前处理步骤。
- 姿态识别状态。
- 舞者检测状态。
- 节拍检测状态。
- 动作节点生成状态。

### 跳转逻辑

- 分析成功且检测到舞者后，进入舞者选择页。
- 分析失败时，允许用户重试或返回首页。
- 用户中途离开后，项目在首页显示为分析中或分析失败状态。

### 页面 4：舞者选择页

### 内容

- 视频预览画面。
- 人物编号或框选提示。
- 目标舞者选择按钮。
- 重新检测入口。

### 跳转逻辑

- 用户选择目标舞者后，App 生成该舞者的练习时间轴。
- 时间轴生成完成后进入练习页。
- 如果用户不满意检测结果，可以重新检测或返回首页。

### 页面 5：练习页

### 内容

- 视频播放器。
- 播放速度控制：0.5x、0.75x、1x。
- 镜像模式。
- 时间轴节点。
- 难点片段标记。
- 循环播放控制。
- 节拍提示。
- 目标舞者轨迹或换位提示。

### 跳转逻辑

- 用户可以从首页直接进入已完成项目的练习页。
- 用户可以点击时间轴节点跳转到指定时间点。
- 用户可以开启片段循环，反复练习某个动作段落。
- 用户可以返回首页选择其他项目。

## 6.2 页面时间逻辑关系

### 首次使用路径

1. 首页：用户看到空状态，点击导入视频。
2. 视频导入页：用户选择本地舞蹈视频，创建学习项目。
3. 视频分析页：App 自动分析视频，生成姿态数据、节拍数据和动作节点。
4. 舞者选择页：用户选择想学习的目标舞者。
5. 练习页：用户开始慢速、镜像、节拍和分段循环练习。

### 回访使用路径

1. 首页：用户打开 App，看到已有学习项目。
2. 如果项目已完成分析，点击后直接进入练习页。
3. 如果项目仍在分析，点击后进入视频分析页查看进度。
4. 如果项目已分析但未选择舞者，点击后进入舞者选择页。
5. 用户练习结束后返回首页，项目保留上次练习状态。

### 单个项目的状态流转

1. Created：视频已导入，学习项目已创建。
2. Analyzing：App 正在分析姿态、节拍和动作节点。
3. NeedsDancerSelection：视频已完成基础分析，等待用户选择目标舞者。
4. ReadyToPractice：目标舞者已选择，练习时间轴已生成。
5. Practicing：用户正在练习页播放、跳转节点或循环片段。

### 练习页内部时间逻辑

- 视频时间轴是练习页的核心时间基准。
- Beat nodes、Key action nodes、Transition nodes 都绑定到同一个视频时间轴。
- 点击节点后，播放器跳转到该节点对应时间。
- 循环练习使用开始时间和结束时间控制片段范围。
- 播放速度只影响播放节奏，不改变节点原始时间。
- 镜像模式只影响画面显示，不改变视频文件和分析数据。
- 队形轨迹和换位提示跟随当前播放时间同步显示。

## 7. AI 分析流程

### 7.1 视频处理流程

1. 用户导入视频。
2. App 抽取视频帧。
3. Vision framework 检测人体姿态。
4. 生成每个舞者的关键点序列。
5. 进行目标舞者跟踪。
6. 分析动作变化。
7. 生成动作节点。
8. 提取音频并分析节拍。
9. 对齐动作节点和节拍节点。
10. 输出练习时间轴。

### 7.2 数据结构示例

```swift
struct DanceNode {
    let id: UUID
    let time: TimeInterval
    let type: DanceNodeType
    let difficulty: DifficultyLevel
    let label: String
}

enum DanceNodeType {
    case beat
    case keyAction
    case transition
}

enum DifficultyLevel {
    case easy
    case medium
    case hard
}
```

## 8. 非功能需求

### 8.1 性能

- 视频播放应流畅。
- 分析过程应提供明确进度。
- 初期允许分析时间较长，但需要避免 App 无响应。
- 大视频应做分段处理或后台处理。

### 8.2 隐私

- 默认本地处理用户视频。
- 未经用户授权不上传视频。
- 清楚说明视频数据用途。
- 订阅或云端 AI 功能上线时，需要单独隐私说明。

### 8.3 兼容性

- 支持近几代 iPhone。
- 优先适配竖屏使用。
- 视频播放区域支持横屏视频。

### 8.4 可用性

- 控制按钮要适合练习时快速点击。
- 练习页避免信息过载。
- 关键功能应能单手操作。
- 时间轴节点要清晰可见。

## 14. 风险与挑战

### 14.1 技术风险

- 多人舞者跟踪难度高。
- 遮挡和换位可能造成目标丢失。
- iPhone 本地视频分析性能有限。
- 节拍检测对复杂音乐可能不够准确。
- AI 动作节点可能和用户理解的舞蹈重点不完全一致。

### 14.2 产品风险

- 用户可能期待过高精度的 AI 拆舞。
- 版权内容处理需要谨慎。
- 免费版和订阅版边界需要设计清楚。
- 初期内容演示质量会影响用户信任。

### 14.3 应对策略

- 初期明确定位为“辅助学习工具”，不是专业老师替代品。
- 支持用户手动调整节点，提高容错。
- 优先支持固定机位、清晰画质的视频。
- 不提供侵权视频下载功能。
- 先把慢速、镜像、分段循环等确定性体验做好。

## 15. 结论

这款 App 的核心不是简单的视频播放器，而是一个面向 K-pop 翻跳学习的 AI 练习工具。MVP 应优先验证三个关键价值：

1. 用户是否能更快找到并跟学目标舞者。
2. 用户是否能通过慢速、镜像、节点和节拍更高效地练习。
3. AI 动作拆解是否能为用户节省真实学习时间。

如果 MVP 能证明这些价值，再逐步扩展到高精度难点检测、队形分析、动作评分和订阅商业化。
